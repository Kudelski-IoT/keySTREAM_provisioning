MPLAB Harmony 3 Integration Guide

Complete guide for integrating keySTREAM Trusted Agent (KTA) library into MPLAB Harmony 3 projects with support for Microchip SAM/PIC32 MCUs.

MPLAB Harmony 3 Overview

Aspect Details
Framework Modular RTOS-less HAL & middleware
Supported MCUs SAM D, E, G, L series + PIC32CM, PIC32MX/MZ
Package Manager Harmony Content Manager (HCM)
Build System MPLAB X IDE with XC32/GCC
Repository GitHub (microchip-pic-avr-solutions)

Installation Steps

Step 1: Create New Harmony Project

1. Open MPLAB X IDE
2. File > New Project > "Microchip Embedded > 32-bit MCU (PIC32/ARM)"
3. Select your target MCU (e.g., ATSAME54P20A)
4. Choose "Start from scratch"
5. Finish project creation

Step 2: Launch Harmony Content Manager

1. Within project: Tools > Embedded > MPLAB Harmony 3 Content Manager
2. Search for desired components:
- "SERCOM" (for UART/SPI/I2C)
- "CRYPTO" (optional, if using SW crypto)
- "TCPIP" (if using Ethernet/WiFi)
3. Click "Finish" to integrate

Step 3: Configure SERCOM for KTA

For I2C (Secure Element):

Setting Value
SERCOM Instance SERCOM0 (or available)
Mode I2C
Speed 100 kHz (standard) or 400 kHz (fast)
Address Length 7-bit
SCL Pin PA23 (example)
SDA Pin PA22 (example)

For UART (Debug/Provisioning):

Setting Value
SERCOM Instance SERCOM1 (or available)
Mode USART
Baud Rate 115200
Data Bits 8
Stop Bits 1
Parity None
TX Pin PA10 (example)
RX Pin PA11 (example)

Step 4: Add KTA Library

1. Copy KTA source files to src/kta/
2. Update project include paths:
- Properties > xc32 > xc32-cpp > Include Directories
- Add: "../src/kta/include"
3. Update Linker Paths if using pre-compiled KTA library

Step 5: Configure KTA HAL

Edit src/config/default/kta/kta_hal.c:

/* Secure Element I2C Interface */
#define KTA_SE_I2C_PORT SERCOM0_I2C_PORT
#define KTA_SE_I2C_SPEED 400000 // 400 kHz
/* UART for debug output */
#define KTA_UART_PORT SERCOM1_UART_PORT
#define KTA_UART_BAUD 115200
/* System Clock */
#define KTA_SYS_CLOCK 120000000 // 120 MHz
/* Tick Timer */
#define KTA_TICK_SOURCE TC0_TIMER_COMPARE

Development Workflow

1. Initialize Application

/* app_init.c */
#include "kta.h"
void APP_Initialize(void) {
KTA_Init(&kta_config);
KTA_SetSecureElement(SE_I2C_PORT);
KTA_EnableDebug(UART_PORT);
}

2. Main Loop

/* app.c */
void APP_Tasks(void) {
switch (app_state) {
case APP_STATE_INIT:
APP_Initialize();
app_state = APP_STATE_IDLE;
break;
case APP_STATE_IDLE:
KTA_Process(); // Non-blocking task
if (KTA_IsProvisioned()) {
app_state = APP_STATE_CONNECTED;
}
break;
case APP_STATE_CONNECTED:
// Send telemetry, handle cloud messages
KTA_PublishMessage(topic, payload);
break;
}
}

3. Provisioning Hook

/* Implement KTA provisioning callback */
void KTA_OnProvisioningComplete(KTA_Device_t *device) {
LED_SetStatus(LED_GREEN, LED_ON);
UART_WriteString("Device provisioned!\r\n");
}

Build & Debug

Building the Project

1. Select project in MPLAB X IDE
2. Clean & Build (Shift+F11)
3. Check console for errors
4. Resolve missing includes or linker errors

Debugging with EDBG

1. Project > Set Configuration > Customize
2. Debug Tab > Debugger: Embedded Debugger (EDBG)
3. Debug Main Project (Ctrl+F5)
4. Use Breakpoints, Watch variables, Memory view

UART Console Output

1. Tools > Serial Monitor (or Tera Term)
2. Set baud: 115200
3. Observe KTA debug output:
- "[KTA] Initializing..."
- "[KTA] SE detected: ATECC608"
- "[KTA] Provisioning token: xxxxxxxx"

Best Practices

Practice Benefit
Use HAL abstractions Code portability across MCUs
Enable KTA debug early Easier troubleshooting
Test UART first Confirms basic HAL setup
Validate SE communication Before provisioning flow
Implement watchdog Protection against hangs
Use DMA for transfers Improves performance
Profile power consumption Optimize sleep/wake cycles

Troubleshooting

Issue Cause Solution
Linker error: undefined kta_init KTA source not linked Add KTA source to project
No UART output SERCOM not configured Check UART pin mapping + clock
SE not detected (I2C) Bad HAL or pin config Verify SCL/SDA pins + pull-ups
Provisioning fails Network or token issue Check cloud backend + token validity
Code size > Flash Too many features enabled Reduce KTA features in kta_config.h
