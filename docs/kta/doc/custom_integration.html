Custom Device and Network Integration

Custom Device and Network Integration

This guide helps you integrate the keySTREAM Trusted Agent (KTA) library into your own hardware platform and network stack. Choose the path that matches your integration scenario.

Choose Your Integration Path

Custom Hardware

Bring your own MCU and secure element. Adapt the KTA library HAL (Hardware Abstraction Layer) to your platform.

Use this if:

  • You have a custom PCB or non-Microchip MCU
  • You need to port the secure element driver (I²C/SPI)
  • You want to use a different SE than ATECC608

What you need to implement:

HAL Layer Purpose Example
Crypto HAL SE communication (sign, verify, key gen) ktaCryptoHal.c
Timer HAL Millisecond tick for timeouts ktaTimerHal.c
Storage HAL Non-volatile storage for KTA state ktaStorageHal.c
Platform HAL Heap allocation, reset, random seed ktaPlatformHal.c

Steps:

  1. Copy include/ and kta/ folders into your project
  2. Implement the 4 HAL files for your platform
  3. Initialize KTA with ktaInit() after your hardware is ready
  4. Call ktaProvisioning() to start device onboarding
  5. Use ktaNotification() in your main loop for keep-alive
/* Minimal KTA integration */
#include "k_kta.h"
void app_init(void) {
// 1. Init your hardware (MCU, SE, network)
platform_init();
// 2. Init KTA library
ktaInit(&config);
// 3. Provision (one-time)
ktaProvisioning(token, token_len);
}
void app_loop(void) {
// 4. Periodic notification (keep-alive with keySTREAM)
ktaNotification();
}

Custom Network Stack

Bring your own transport layer (Wi-Fi, Cellular, Ethernet, LoRa, etc.). Adapt the KTA communication interface to your network.

Use this if:

  • You have a custom network stack (not Harmony NET)
  • You're using cellular (4G/5G/NB-IoT), LoRa, or satellite
  • You want to use a different MQTT/TLS library

What you need to implement:

Comm Layer Purpose Example
TLS HAL TLS session (connect, send, receive, close) ktaTlsHal.c
MQTT HAL MQTT publish/subscribe (or use built-in) ktaMqttHal.c
DNS HAL Hostname resolution ktaDnsHal.c

Steps:

  1. Integrate your TLS library (mbedTLS, wolfSSL, OpenSSL, etc.)
  2. Implement ktaTlsHal.c with connect/send/recv/close
  3. Configure the keySTREAM endpoint in ktaConfig.h
  4. The KTA library handles MQTT framing internally
  5. Test connectivity with the keySTREAM sandbox
/* TLS HAL interface - implement for your stack */
TKStatus ktaTlsConnect(const char* host, uint16_t port);
TKStatus ktaTlsSend(const uint8_t* data, size_t len);
TKStatus ktaTlsRecv(uint8_t* buffer, size_t* len, uint32_t timeout_ms);
TKStatus ktaTlsDisconnect(void);
